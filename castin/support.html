System Requirements

  • macOS 10.15 (Catalina) or later
  • Intel or Apple Silicon processor
  • At least 100MB free disk space
  • Network connection for DLNA functionality

First-Time Setup

  1. Download and install Castin from the official source
  2. Launch the application
  3. Allow necessary network permissions when prompted

  5. The DLNA service starts automatically

üîß Common Issues & Solutions

Service Won't Start

Solution:
  • Check if another application is using port 8199
  • Ensure macOS firewall allows Castin connections
  • Restart the application
  • Check your network connection

Devices Cannot Discover Castin

Solution:
  • Ensure all devices are on the same local network
  • Check your router's multicast settings
  • Verify DLNA/UPnP is enabled on your casting device
  • Restart your router if necessary

Streaming Issues

Common Problems:
  • Buffering: Check your network bandwidth and stability
  • Connection Drops: Verify network stability and device proximity
  • No Audio/Video: Ensure media format compatibility
  • High Latency: Check network congestion and server load

Download Problems

Troubleshooting Steps:
  • Verify M3U8 URL is accessible and valid
  • Check available disk space in your Documents folder
  • Ensure internet connection is stable
  • Try downloading a different M3U8 stream for testing

üìã Frequently Asked Questions

Q: What is Castin?

A: Castin is a macOS application that acts as a DLNA media receiver, allowing you to cast media content from your mobile devices or computers to your Mac.

Q: How do I cast content to Castin?

A: Use any DLNA/UPnP compatible casting app on your device, search for "Castin" or your custom device name, and select it as the casting target.

Q: What media formats are supported?

A: Castin supports most common media formats through FFmpeg integration, including MP4, MKV, AVI, MOV, and various streaming formats.

Q: Where are downloaded files stored?

A: Downloaded files are saved in: ~/Documents/CastinDownloads/

Q: Can I change the default port?

A: Currently, Castin uses port 8199 by default. Port customization may be added in future versions.

Q: Is my data private?

A: Yes! Castin processes all data locally on your device. No personal information or media content is transmitted to external servers.

üîç Technical Details

Network Requirements

  • Protocol: DLNA/UPnP over HTTP
  • Port: 8199 (default)
  • Discovery: SSDP (Simple Service Discovery Protocol)
  • Transport: TCP for control, UDP for discovery

FFmpeg Integration

Castin uses FFmpeg for media processing. The application includes a properly signed FFmpeg framework that handles:

  • Media format detection and conversion
  • M3U8 stream downloading
  • Video and audio codec support
  • Stream analysis and processing

File Locations

Application Support: ~/Library/Application Support/Castin/ Preferences: ~/Library/Preferences/com.yourcompany.Castin.plist Downloads: ~/Documents/CastinDownloads/ Logs: ~/Library/Logs/Castin/

Debug Mode

To enable debug logging:

  1. Open Terminal
  2. Launch Castin with debug flag: open -a Castin --args -debug
  3. Check Console.app for Castin logs

Manual Service Restart

If the service becomes unresponsive:
  1. Quit Castin completely
  2. Wait 10 seconds
  3. Relaunch the application
  4. The service should restart automatically
